# Shift-Meeting-Scheduler-Optimizer
一個專門為解決產線排班與會議衝突開發的自動化優化工具
# 📊 半導體廠排班與會議時程優化器 (Shift & Meeting Scheduler Optimizer)

## 📌 專案動機 (Why)
在大型半導體晶圓廠中，工程師多採取「四班三輪」或複雜的排班機制。傳統人工排班不僅耗時，更常發生「安排會議時，關鍵工程師正在輪休或值夜班」的衝突狀況。
本專案透過自動化邏輯，能自動媒合員工班表與會議時間，**預計能降低 80% 的會議排程衝突率，提升跨部門協作效率。**

## 🛠 技術棧 (Tech Stack)
- **語言**: Python 3.x
- **核心邏輯**: 條件約束檢查、自動化排程演算法
- **工具**: GitHub Actions (未來擴調), Google Colab

## 🗺 系統架構 (System Architecture)
以下為本排班優化器的核心邏輯：
[員工班表輸入] ──> [衝突檢查核心] ──> [自動產生最佳會議時間]
                             │
                             └──> [觸發異常警告 (人數不足)]

執行 `scheduler_optimizer.py` 後，系統會自動輸出如下結果：
- 自動辨識出「交接溝通會議」出席人數不足。
- 系統跳出 ⚠️ 警告，提醒管理者調整時間。
